数据采集指南

这一章保留项目原有的数据采集文档，说明为什么使用 TikHub、采集层如何组织、数据写到哪里，以及哪些命令负责把数据准备成站点可用格式。

为什么使用 TikHub

  • 一个 API key 可以覆盖多个社交平台。
  • Instagram 相关接口已经覆盖资料、帖子分页、评论、回复和搜索。
  • 规范化、存储、导出和文档发布逻辑仍然由仓库自己维护。

API key 配置

  1. 注册或登录 TikHub 账号。
  2. 在 TikHub dashboard 中生成 API key。
  3. 把 key 写入本地 .env。
TIKHUB_API_KEY=your_real_api_key

采集架构

  • src/collect/：client、采集接口、Instagram 实现、CLI 和导出逻辑。
  • data/collect/：账号快照、post/reel 条目记录，以及对应的原始 API 返回。
  • data/dashboard/：规范的 dashboard JSON 数据。
  • docs/data/：发布站点使用的镜像副本。

当前主要命令入口包括：

  • python3 -m src.collect collect-account ...
  • python3 -m src.collect export-dashboard --input data/collect
  • python3 -m src.collect sync-docs-data
  • python3 -m src.collect serve

本地运行方式

python3 -m src.collect collect-account \
  --platform instagram \
  --username nasa \
  --max-posts 5

python3 -m src.collect collect-account \
  --platform instagram \
  --username nasa \
  --include-comments \
  --max-posts 3 \
  --max-comment-pages 2

python3 -m src.collect export-dashboard --input data/collect
python3 -m src.collect sync-docs-data
python3 -m src.collect serve

写入 data/ 的内容

采集数据现在按账号组织，再按 post 或 reel 条目拆分：

data/collect/<platform>/<account-slug>/
  account.json
  raw/account/profile.json
  media/account/profile.jpg
  posts/<post-code-or-id>/item.json
  reels/<reel-code-or-id>/item.json
  • account.json：规范化后的账号资料、已保存条目的聚合指标、请求元数据，以及账号级提取时间戳。
  • posts/*/item.json 与 reels/*/item.json：每个条目一份规范化记录，包含 comments、replies、请求元数据、本地媒体路径、完整的图片/视频资产列表，以及条目提取时间戳。
  • raw/account/profile.json：账号资料原始返回。
  • raw/account_posts/*.json：账号 feed 分页原始返回。
  • posts/*/raw/detail.json：条目详情原始返回。
  • posts/*/raw/comments/*.json 与 reels/*/raw/comments/*.json：各条目的评论分页原始返回。
  • posts/*/raw/replies/*.json 与 reels/*/raw/replies/*.json：各条目的回复分页原始返回。
  • media/account/*：下载后的账号头像等图片。
  • posts/*/media/* 与 reels/*/media/*：按条目保存的媒体文件，包括 reel 视频和混合轮播中的全部媒体资源。

Dashboard 导出约定

导出器会把采集得到的账号记录和条目记录转换成供站点使用的静态 JSON，规范文件包括：

  • data/dashboard/site-summary.json
  • data/dashboard/accounts.json
  • data/dashboard/posts.json
  • data/dashboard/hashtags.json
  • data/dashboard/engagement-timeseries.json

随后的 sync-docs-data 命令会把这些文件复制到 docs/data/， 供 GitHub Pages 发布后的 HTML 页面读取。

计费与限流说明

TikHub 采用按量付费。根据 2026 年 4 月 19 日公开页面，本项目使用的若干 Instagram 接口标注为 0.002 USD/request，但价格随时可能变化，正式大规模采集前应重新确认。

  • TikHub pricing
  • TikHub docs

错误处理

  • 401：API key 缺失、无效或已过期。
  • 402：余额不足，无法访问付费接口。
  • 429：达到限流阈值，client 会执行退避重试。
  • 5xx：上游临时故障会自动重试。
