GitHub Pages 与站点数据配置

请使用默认分支并发布 /docs 文件夹。站点入口文件现在是 docs/index.html。

GitHub Pages 设置步骤

  1. 先把仓库推送到 GitHub。
  2. 打开仓库页面。
  3. 进入 Settings -> Pages。
  4. 在 Build and deployment 中选择 Deploy from a branch。
  5. 选择默认分支，通常是 main。
  6. 选择 /docs 文件夹。
  7. 保存并等待部署完成。

预计访问地址

https://<your-github-username>.github.io/<repository-name>/

数据刷新流程

站点实际发布的是 docs/ 中的文件，但规范的 dashboard 数据保存在 data/dashboard/。导出后还需要同步发布副本： 

python3 -m src.collect export-dashboard --input data/collect
python3 -m src.collect sync-docs-data

# 或者
make docs-data

重要说明

  • 文档站已经改成 HTML，因此页面结构、样式和脚本都直接维护在 docs/ 内。
  • 发布站点读取的是 docs/data/*.json。
  • 当数据约定发生变化时，应同时更新导出器、同步命令和文档页面。
