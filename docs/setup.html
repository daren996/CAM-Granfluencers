Setup

GitHub Pages and docs data setup

Use the default branch and publish the /docs folder. The site entry file is now docs/index.html.

GitHub Pages setup

  1. Push the repository to GitHub.
  2. Open the repository page.
  3. Go to Settings -> Pages.
  4. Under Build and deployment, choose Deploy from a branch.
  5. Select the default branch, usually main.
  6. Select the /docs folder.
  7. Save and wait for the deployment to finish.

Expected URL

https://<your-github-username>.github.io/<repository-name>/

Data refresh workflow

The site publishes files from docs/, but canonical dashboard data lives in data/dashboard/. Refresh the site copy after export: 

python3 -m src.collect export-dashboard --input data/collect
python3 -m src.collect sync-docs-data

# or
make docs-data

Important notes

  • The docs site is HTML-based, so page structure, styling, and scripts live directly in docs/.
  • The published site reads docs/data/*.json.
  • When data contracts change, update the exporter, sync command, and docs pages together.
