项目地图与 Wiki 章节

这一页是仓库的 Wiki 首页，负责整理项目结构、双语文档站规则，并作为各个 Wiki 子页的总入口，例如数据采集指南。

Wiki 章节

  • 项目地图与规则：仓库结构、文档工作流和发布约定。
  • 数据采集指南：TikHub 配置、本地命令、输出结构与导出流程。

仓库结构

docs/
由 GitHub Pages 发布的双语 HTML 站点。它是项目面向使用者的流程、导航和页面结构的规范来源。
src/collect/
Python 数据采集层，负责 TikHub 访问、规范化处理、导出以及本地数据准备命令。
data/
项目的规范数据目录。按账号组织的采集快照与 post/reel 条目保存在 data/collect/，面向站点的数据保存在 data/dashboard/。

文档优先规则

  • docs/ 已经改成 HTML 站点，新文档页面应继续使用 HTML，而不是 Markdown。
  • 每个页面都必须保留英文版与简体中文版。
  • 每对页面都要在顶部提供明显的语言切换入口。
  • 主页为 docs/index.html，主导航结构应保持稳定。
  • 如果代码行为和文档站承诺不一致，应优先修复代码，或者在同一次修改中同步更新两者。

数据发布流程

  1. 先把采集结果写入 data/collect/。
  2. 再把 dashboard JSON 导出到 data/dashboard/。
  3. 把发布所需副本同步到 docs/data/。
  4. 需要发布时，同时提交文档改动和同步后的站点数据。
python3 -m src.collect collect-account --platform instagram --username nasa --max-posts 5
python3 -m src.collect export-dashboard --input data/collect
python3 -m src.collect sync-docs-data

如果需要完整的采集章节，请打开 数据采集指南。

为什么改成 HTML

站点改成 HTML 后，GitHub Pages 的输出会更稳定，也更方便继续扩展导航栏、自定义布局、脚本逻辑，以及未来的 TypeScript 或可视化层，而不再受 Markdown 默认渲染方式的限制。
