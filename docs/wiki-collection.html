Collection guide

This chapter preserves the project's collection documentation: why TikHub is used, how the collector is structured, what files are written, and which commands prepare data for the site.

Why TikHub

  • One API key can cover multiple social platforms.
  • Instagram endpoints already cover profile lookup, post pagination, comments, replies, and search.
  • The repository keeps normalization, storage, export, and docs publication logic in-house.

API key setup

  1. Create or sign in to a TikHub account.
  2. Generate an API key from the TikHub dashboard.
  3. Store it in the local .env file.
TIKHUB_API_KEY=your_real_api_key

Collector architecture

  • src/collect/: client, collector interfaces, Instagram implementation, CLI, and export logic.
  • data/collect/: account snapshots, post/reel item records, and raw API responses.
  • data/dashboard/: canonical dashboard-facing JSON.
  • docs/data/: published mirror used by the HTML site.

The public CLI entrypoints are:

  • python3 -m src.collect collect-account ...
  • python3 -m src.collect export-dashboard --input data/collect
  • python3 -m src.collect sync-docs-data
  • python3 -m src.collect serve

Local usage

python3 -m src.collect collect-account \
  --platform instagram \
  --username nasa \
  --max-posts 5

python3 -m src.collect collect-account \
  --platform instagram \
  --username nasa \
  --include-comments \
  --max-posts 3 \
  --max-comment-pages 2

python3 -m src.collect export-dashboard --input data/collect
python3 -m src.collect sync-docs-data
python3 -m src.collect serve

Data written to data/

Collected data is organized by account, then by post or reel item:

data/collect/<platform>/<account-slug>/
  account.json
  raw/account/profile.json
  media/account/profile.jpg
  posts/<post-code-or-id>/item.json
  reels/<reel-code-or-id>/item.json
  • account.json: normalized account profile, stored item metrics, request metadata, and the account-level extraction timestamp.
  • posts/*/item.json and reels/*/item.json: one normalized record per collected item, including comments, replies, request metadata, local media paths, full image/video asset lists, and the item extraction timestamp.
  • raw/account/profile.json: raw profile response.
  • raw/account_posts/*.json: raw account feed page responses.
  • posts/*/raw/detail.json: raw item detail response.
  • posts/*/raw/comments/*.json and reels/*/raw/comments/*.json: raw comment page responses for each item.
  • posts/*/raw/replies/*.json and reels/*/raw/replies/*.json: raw reply page responses for each item.
  • media/account/*: downloaded account profile images.
  • posts/*/media/* and reels/*/media/*: downloaded media files saved per collected item, including reel videos and every asset inside mixed carousels.

Dashboard export contract

The exporter transforms collected account and item records into static JSON used by the site. The canonical files are:

  • data/dashboard/site-summary.json
  • data/dashboard/accounts.json
  • data/dashboard/posts.json
  • data/dashboard/comments.json
  • data/dashboard/hashtags.json
  • data/dashboard/engagement-timeseries.json
  • data/dashboard/collection-tree.json

The sync-docs-data command copies those files into docs/data/ for GitHub Pages publishing.

Cost and rate limit notes

TikHub uses pay-as-you-go pricing. Several Instagram endpoints referenced by this project were documented at 0.002 USD/request on April 19, 2026, but pricing can change. Re-check before large runs.

  • TikHub pricing
  • TikHub docs

Error handling

  • 401: missing, invalid, or expired API key.
  • 402: insufficient balance for a paid endpoint.
  • 429: rate limit hit; the client retries with backoff.
  • 5xx: transient upstream failures are retried automatically.
