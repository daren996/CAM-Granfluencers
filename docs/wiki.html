Project map and wiki chapters

This page acts as the wiki hub for the repository. It records the structure of the project, the rules that keep the bilingual docs site aligned, and links to dedicated wiki chapters such as the collection guide.

Repository map

docs/
Bilingual HTML site published through GitHub Pages. This is the source of truth for project-facing workflows, page structure, and navigation.
src/collect/
Python collection layer for TikHub access, normalization, export, and local data preparation commands.
data/
Canonical local data storage. Collected account snapshots and post/reel items live in data/collect/, and dashboard-facing outputs live in data/dashboard/.

Docs-first rules

  • docs/ is HTML-based. New docs pages should be added as HTML, not Markdown.
  • Every docs page must have an English and Simplified Chinese pair.
  • Each pair must expose a visible language switch at the top of the page.
  • The homepage is docs/index.html and should keep the main navigation intact.
  • If code behavior conflicts with what the docs site promises, fix the code or update both together.

Data publication workflow

  1. Run the collector into data/collect/.
  2. Export dashboard JSON into data/dashboard/.
  3. Sync the publishable copy into docs/data/.
  4. Commit both the docs changes and the synchronized site data when needed.
python3 -m src.collect collect-account --platform instagram --username nasa --max-posts 5
python3 -m src.collect export-dashboard --input data/collect
python3 -m src.collect sync-docs-data

For the full collection chapter, open Collection guide.

Why HTML-based docs

Moving the site to HTML keeps the GitHub Pages output predictable and makes it easier to add custom navigation, layout, scripts, and future TypeScript or visualization layers without relying on Markdown rendering details.
